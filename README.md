# Workshop: Contact App

## MVC Design Pattern Explanation:
As a manager, you use the MVC (Model–View–Controller) pattern to explain the structural roles of
the project before development starts. The structure is divided into three parts, the model,
the controller and the view. The model, which contains information or represents real world objects.
The controller, which manages the inputs from the user, access the information in the model and send
the data to the view. The view, which presents the interface and the menu of the application.
It also displays the information that the view receives from the controller so that the user can see it.
